项目概览
本项目是一个纯前端网页应用，用于实时分析 2026 年 FIFA 世界杯小组赛的出线形势。用户输入比赛比分后，系统立即穷举所有剩余比赛的可能结果，计算每支球队的出线概率，并通过颜色系统直观展示。
技术特点：纯浏览器端计算 | 零 API 依赖 | 离线可用 | 单页应用 | 响应式设计

3.2 js/colorSystem.js —— 颜色状态系统
文件路径：
D:\llm\yz_football\js\colorSystem.js
作用：将出线概率百分比映射为颜色和状态标签。完全由数学结果驱动，无需人工干预。
阈值配置：
= 100%  →  ✅ 已锁定（绿色 #00C853）
> 66.7% →  🟢 稳（偏绿 #4CAF50）
33.3%~66.7% →  🟡 有戏（中间色 #FF9800）
< 33.3% 且 > 0% →  ⚫ 悬（偏黑 #757575）
= 0% →  💀 已出局（纯黑 #212121）
特殊状态：
• 🥇 已锁定头名（所有穷举场景均为第一）→ 金色 #FFD700
• 🥈 已锁定第二（所有穷举场景均为第二）→ 银色 #C0C0C0
• 🔒 已锁定前二（确保直接出线，但不一定是第几）→ 绿色
提供方法：
getStatus(rate, extras)     — 返回 {label, color, bgColor, cssClass, icon, level}
getProgressGradient(rate)   — 返回进度条渐变色
getTextColor(rate)          — 返回对应文字颜色
3.3 js/rules.js —— FIFA 排名规则
文件路径：
D:\llm\yz_football\js\rules.js
作用：实现 FIFA 官方小组赛排名规则，包括积分计算、同分 Tie-breaking、跨组第三名比较。
核心方法：
getMatchResult(match)              — 计算单场比赛积分贡献
computeStandings(teams, matches)    — 计算完整积分榜并排序
compareTeams(a, b, allStats)        — 两支球队排名比较（7层优先级）
├── compareTwoTied(a, b)          — 两队同分：净胜球→进球→H2H
└── compareMultiTied(a, b, ...)   — 多队同分：构建H2H小联赛
buildH2HMiniLeague(tiedTeams, ...)  — 构建仅涉及TIE队的相互战绩表
rankThirdPlaces(thirdPlaceTeams)    — 跨组12选8排序
FIFA 排名优先级（依次比较）：
① 积分 → ② 净胜球（全量）→ ③ 总进球（全量）→ ④ 相互积分 → ⑤ 相互净胜球 → ⑥ 相互进球 → ⑦ 公平竞赛（简化）→ ⑧ 抽签（标记TIE）
技术细节：
• 多队同分时按 FIFA 规则构建"小联赛"：仅计算 TIE 队之间的相互比赛
• 公平竞赛积分和抽签在当前版本中简化处理，标记为 TIE 供用户关注
3.4 js/engine.js —— 核心计算引擎
文件路径：
D:\llm\yz_football\js\engine.js
作用：整个应用的计算核心。穷举/采样所有剩余比赛的比分可能性，统计每队在各场景下的排名结果，计算出线概率。
核心方法：
generateAllScores()          — 生成0:0~4:4共25种比分组合
cartesianProduct(arrays)      — 笛卡尔积（组合穷举）
analyzeGroup(group, ...)      — ★ 核心函数，分析单个小组出线形势
buildFullMatches(...)         — 合并已完赛+穷举比分为完整比赛数据
sampleScenarios(...)          — 蒙特卡洛随机采样（全6场未赛时用）
analyzeAllGroups(...)         — 全局跨组分析（含第三名估算）
estimateThirdPlaceQualification(...) — 基于历史模型估算第三名晋级概率
getStrategy(remainingCount)   — 根据剩余场次选择穷举策略
穷举策略：
剩余 ≤4 场 → 完整穷举（25^N，最大 390,625 场景，<3ms）
剩余 = 5 场 → 完整穷举（25^5 ≈ 9.7M 场景，约 5ms，可接受）
剩余 = 6 场 → 蒙特卡洛采样 100,000 次（概率误差 < ±1.5%）
第三名晋级估算模型（基于历史统计）：
6.0分+ →  96%（几乎锁定）     5.0分+ →  85%（大概率）
4.5分+ →  70%（较大概率）     4.0分+ →  60% or 40%（取决于净胜球正负）
3.0分+ →  20% or 5%（取决于净胜球正负）    <3.0分 →  1%
技术细节：
• 多队同分时按 FIFA 规则构建"小联赛"：仅计算 TIE 队之间的相互比赛
• 公平竞赛积分和抽签在当前版本中简化处理，标记为 TIE 供用户关注

六、数据流说明
以下是用户操作触发一次完整计算的数据流：
用户点击"保存比分"
│
▼
main.js: saveMatchScore()
│  更新 allMatches[group][i].homeScore / awayScore
│  调用 saveScores() → localStorage.setItem(...)
│  调用 recalculateAll()
▼
main.js: recalculateAll()
│  for each group (A~L):
│    engine.js: analyzeGroup(group, matches, teamMap)
│      │  分离 playedMatches / remainingMatches
│      │  根据剩余场数选择完整穷举或蒙特卡洛采样
│      │  for each scenario:
│      │    rules.js: computeStandings() → 排名
│      │    统计：前2/第3/第4/第一/第二 次数
│      │    统计：积分/净胜球/进球 累计
│      ▼
│    返回 {group, totalScenarios, results, standings}
│    colorSystem.js: getStatus() → 颜色标签
│
│  engine.js: analyzeAllGroups(groupResults) → 全局比较
│    对每队计算第三名晋级概率 + 总出线概率
▼
renderer.js: renderAll(state)
│  renderGroupNav()    → 顶部小组按钮
│  renderStandings()   → 积分榜 + 概率进度条
│  renderMatches()     → 比赛卡片（3轮）
│  renderSummary()     → 形势概览
▼
用户看到更新后的界面

八、维护与更新指南
如何更新球队名单：
编辑 js/data.js 中的 EMBEDDED_TEAMS 数组，按相同格式替换球队信息。同步更新 data/teams.json。
如何更新比赛赛程：
编辑 js/data.js 中的 EMBEDDED_MATCHES 数组，按相同格式更新比赛对阵。同步更新 data/matches.json。
如何调整颜色阈值：
编辑 js/colorSystem.js 中的 thresholds 对象，修改 safe / hopeful 等阈值百分比。
如何调整比分穷举范围：
编辑 js/engine.js 中的 scoreRange 数组，例如改为 [0,1,2,3] 缩小范围（16种/场），提高计算速度。
如何调整蒙特卡洛采样数：
编辑 js/engine.js 中的 monteCarloSamples 值。数值越大越精确，但计算时间越长。推荐 50000~200000。
